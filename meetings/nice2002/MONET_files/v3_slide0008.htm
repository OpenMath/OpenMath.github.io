An “Explanation” Ontology
Ø Computations done via MONET could involve
many services, algorithms, machines …
Ø No unified tracing/debugging facility practical
Ø Most basic explanation of how an answer was
arrived at is provided by keeping an audit trail
Ø Services which support explanation facility can
provide extra feedback and justification
Ø Information presented to user rather like a
proof
Ø Can construct tools which replay parts of
computation, interact with input to and output
from services dynamically, change choice of
service or service behaviour etc.
