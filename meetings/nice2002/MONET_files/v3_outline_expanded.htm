MONET Infrastructure
Brokers
register local services
communicate with other brokers
match a query to one or more services
fuzzy matching versus exact matching
Object Manager
refer to objects via URIs
objects may be persistent, exist inside a running service, be stored by a cluster of services etc.
allow access permissions to be set (e.g. for collaborative working)
Explanation and Debugging Tool

An “Explanation” Ontology
Computations done via MONET could involve many services, algorithms, machines …
No unified tracing/debugging facility practical
Most basic explanation of how an answer was arrived at is provided by keeping an audit trail
Services which support explanation facility can provide extra feedback and justification
Information presented to user rather like a proof
Can construct tools which replay parts of computation, interact with input to and output from services dynamically, change choice of service or service behaviour etc.
