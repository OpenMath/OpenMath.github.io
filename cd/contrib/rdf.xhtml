This CD defines symbols for basic integration of RDF data into OpenMath. It should serve as a starting point for future integration scenarios of Linked Data and mathematics. Author: Ken Wenzel, Fraunhofer IWU

resourceset

Role:
application
Description:

This symbol represents an unary construction function for constructing a set of RDF resources. It takes one string argument representing a Manchester Syntax description as described by http://www.w3.org/TR/owl2-manchester-syntax/#Descriptions in order to construct a set of RDF resources.

Please note that it may also be possible to represent the class description by using OpenMath set operations: rdf.resourceset( set1.intersect( rdf.resourceset("foaf:Person"), set1.suchthat(rdf.resourceset("rdfs:Resource"), fns1.lambda[$r -> set1.size(rdf.valueset("foaf:age", $r) > 0)]) ) )

Example:
An example to show the representation of the class of FOAF persons with a known age. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMA>
    <OMS cd="rdf" name="resourceset"/>
    <OMSTR>foaf:Person and foaf:age some rdfs:Literal</OMSTR>
  </OMA>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML"><apply><csymbol cd="rdf">resourceset</csymbol><cs>foaf:Person and foaf:age some rdfs:Literal</cs></apply></math>
resourceset ( "foaf:Person and foaf:age some rdfs:Literal" )
rdf.resourceset("foaf:Person and foaf:age some rdfs:Literal")
resourceset ⁡ ( foaf:Person and foaf:age some rdfs:Literal )

prefixes

Role:
semantic-attribution
Description:

A symbol to be used within an OpenMath attribute to specify one or more RDF namespace prefixes. The annotation value should be a set1.set of pairs of strings (prefix name, namespace URI) constructed with the prefix symbol.

prefix

Role:
application
Description:

A symbol to be used as the head of the OpenMath application to construct a prefix mapping that can be used as a value of the prefixes attribution. The two arguments of this function should be OpenMath strings representing in order, the prefix and the corresponding namespace URI.

Example:
The prefix mappings rdfs:<http://www.w3.org/2000/01/rdf-schema#> and foaf:<http://xmlns.com/foaf/0.1/> for a rdf.resourceset may be represented by: 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMATTR>
    <OMATP>
      <OMS cd="rdf" name="prefixes"/>
      <OMA>
        <OMS cd="set1" name="set"/>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>rdfs</OMSTR>
          <OMSTR>http://www.w3.org/2000/01/rdf-schema#</OMSTR>
        </OMA>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>foaf</OMSTR>
          <OMSTR>http://xmlns.com/foaf/0.1/</OMSTR>
        </OMA>
      </OMA>
    </OMATP>
    <OMA>
      <OMS cd="rdf" name="resourceset"/>
      <OMSTR>foaf:Person and foaf:age some rdfs:Literal</OMSTR>
    </OMA>
  </OMATTR>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics>
  <apply><csymbol cd="rdf">resourceset</csymbol><cs>foaf:Person and foaf:age some rdfs:Literal</cs></apply>
  <annotation-xml cd="rdf" encoding="prefixes">
   <apply><csymbol cd="set1">set</csymbol>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>rdfs</cs><cs>http://www.w3.org/2000/01/rdf-schema#</cs></apply>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>foaf</cs><cs>http://xmlns.com/foaf/0.1/</cs></apply>
   </apply>
  </annotation-xml>
 </semantics>
</math>
Attrib([ prefixes set (prefix ( "rdfs" , "http://www.w3.org/2000/01/rdf-schema#" ) , prefix ( "foaf" , "http://xmlns.com/foaf/0.1/" ) ) ], resourceset ( "foaf:Person and foaf:age some rdfs:Literal" ) )
rdf.resourceset("foaf:Person and foaf:age some rdfs:Literal"){rdf.prefixes -> {rdf.prefix("rdfs", "http://www.w3.org/2000/01/rdf-schema#") , rdf.prefix("foaf", "http://xmlns.com/foaf/0.1/")}}
resourceset ⁡ ( foaf:Person and foaf:age some rdfs:Literal )

resource

Role:
application
Description:

This symbol represents an unary construction function for representing a specific RDF resource. It takes one string argument denoting an IRI reference as prefixed name in the form "prefix:resourceName" or as a full IRI in the form ">IRI<".

Example:
An example to show the representation of the resource http://example.org/persons#Alice using a full IRI reference. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMA>
    <OMS cd="rdf" name="resource"/>
    <OMSTR>&lt;http://example.org/persons#Alice&gt;</OMSTR>
  </OMA>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML"><apply><csymbol cd="rdf">resource</csymbol><cs>&lt;http://example.org/persons#Alice&gt;</cs></apply></math>
resource ( "<http://example.org/persons#Alice>" )
rdf.resource("<http://example.org/persons#Alice>")
resource ⁡ ( <http://example.org/persons#Alice> )
Example:
An example to show the representation of the resource http://example.org/persons#Alice using the prefixed name example:Alice. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMATTR>
    <OMATP>
      <OMS cd="rdf" name="prefixes"/>
      <OMA>
        <OMS cd="set1" name="set"/>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>example</OMSTR>
          <OMSTR>http://example.org/persons#</OMSTR>
        </OMA>
      </OMA>
    </OMATP>
    <OMA>
      <OMS cd="rdf" name="resource"/>
      <OMSTR>example:Alice</OMSTR>
    </OMA>
  </OMATTR>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics>
  <apply><csymbol cd="rdf">resource</csymbol><cs>example:Alice</cs></apply>
  <annotation-xml cd="rdf" encoding="prefixes">
   <apply><csymbol cd="set1">set</csymbol>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>example</cs><cs>http://example.org/persons#</cs></apply>
   </apply>
  </annotation-xml>
 </semantics>
</math>
Attrib([ prefixes set (prefix ( "example" , "http://example.org/persons#" ) ) ], resource ( "example:Alice" ) )
rdf.resource("example:Alice"){rdf.prefixes -> {rdf.prefix("example", "http://example.org/persons#")}}
resource ⁡ ( example:Alice )

value

Role:
application
Description:

This symbol represents a function for accessing the value of an RDF property. It takes two arguments, a string denoting the property and an object denoting the RDF resource whose property value should be retrieved.

Example:
An example that retrieves the property value for http://xmlns.com/foaf/0.1/age of the resource http://example.org/persons#Alice. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMATTR>
    <OMATP>
      <OMS cd="rdf" name="prefixes"/>
      <OMA>
        <OMS cd="set1" name="set"/>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>example</OMSTR>
          <OMSTR>http://example.org/persons#</OMSTR>
        </OMA>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>foaf</OMSTR>
          <OMSTR>http://xmlns.com/foaf/0.1/</OMSTR>
        </OMA>
      </OMA>
    </OMATP>
    <OMA>
      <OMS cd="rdf" name="value"/>
      <OMSTR>foaf:age</OMSTR>
      <OMA>
        <OMS cd="rdf" name="resource"/>
        <OMSTR>example:Alice</OMSTR>
      </OMA>
    </OMA>
  </OMATTR>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics>
  <apply><csymbol cd="rdf">value</csymbol>
   <cs>foaf:age</cs>
   <apply><csymbol cd="rdf">resource</csymbol><cs>example:Alice</cs></apply>
  </apply>
  <annotation-xml cd="rdf" encoding="prefixes">
   <apply><csymbol cd="set1">set</csymbol>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>example</cs><cs>http://example.org/persons#</cs></apply>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>foaf</cs><cs>http://xmlns.com/foaf/0.1/</cs></apply>
   </apply>
  </annotation-xml>
 </semantics>
</math>
Attrib([ prefixes set (prefix ( "example" , "http://example.org/persons#" ) , prefix ( "foaf" , "http://xmlns.com/foaf/0.1/" ) ) ], value ( "foaf:age" , resource ( "example:Alice" ) ) )
rdf.value("foaf:age", rdf.resource("example:Alice")){rdf.prefixes -> {rdf.prefix("example", "http://example.org/persons#") , rdf.prefix("foaf", "http://xmlns.com/foaf/0.1/")}}
value ⁡ ( foaf:age , resource ⁡ ( example:Alice ) )

valueset

Role:
application
Description:

This symbol represents a function for accessing all values of multivalued RDF property. It takes two arguments, a string denoting the property and an object denoting the RDF resource whose property values should be retrieved.

Example:
An example that retrieves the property values for http://xmlns.com/foaf/0.1/knows of the resource http://example.org/persons#Alice and hence all persons known by Alice. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMATTR>
    <OMATP>
      <OMS cd="rdf" name="prefixes"/>
      <OMA>
        <OMS cd="set1" name="set"/>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>example</OMSTR>
          <OMSTR>http://example.org/persons#</OMSTR>
        </OMA>
        <OMA>
          <OMS cd="rdf" name="prefix"/>
          <OMSTR>foaf</OMSTR>
          <OMSTR>http://xmlns.com/foaf/0.1/</OMSTR>
        </OMA>
      </OMA>
    </OMATP>
    <OMA>
      <OMS cd="set1" name="in"/>
      <OMV name="p"/>
      <OMA>
        <OMS cd="rdf" name="valueset"/>
        <OMSTR>foaf:knows</OMSTR>
        <OMA>
          <OMS cd="rdf" name="resource"/>
          <OMSTR>example:Alice</OMSTR>
        </OMA>
      </OMA>
    </OMA>
  </OMATTR>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics>
  <apply><csymbol cd="set1">in</csymbol>
   <ci>p</ci>
   <apply><csymbol cd="rdf">valueset</csymbol>
    <cs>foaf:knows</cs>
    <apply><csymbol cd="rdf">resource</csymbol><cs>example:Alice</cs></apply>
   </apply>
  </apply>
  <annotation-xml cd="rdf" encoding="prefixes">
   <apply><csymbol cd="set1">set</csymbol>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>example</cs><cs>http://example.org/persons#</cs></apply>
    <apply><csymbol cd="rdf">prefix</csymbol><cs>foaf</cs><cs>http://xmlns.com/foaf/0.1/</cs></apply>
   </apply>
  </annotation-xml>
 </semantics>
</math>
Attrib([ prefixes set (prefix ( "example" , "http://example.org/persons#" ) , prefix ( "foaf" , "http://xmlns.com/foaf/0.1/" ) ) ], in ( p, valueset ( "foaf:knows" , resource ( "example:Alice" ) ) ) )
set1.in($p, rdf.valueset("foaf:knows", rdf.resource("example:Alice"))){rdf.prefixes -> {rdf.prefix("example", "http://example.org/persons#") , rdf.prefix("foaf", "http://xmlns.com/foaf/0.1/")}}
p ∈ valueset ⁡ ( foaf:knows , resource ⁡ ( example:Alice ) )

literal_type

Role:
semantic-attribution
Description:

A symbol to be used within an OpenMath attribute to specify the type of an RDF literal which is represented as an OpenMath object. The annotation value should be an rdf.resource.

Example:
An example to show the representation of the typed RDF literal "This is a string."^^<http://www.w3.org/2001/XMLSchema#string> using a full IRI reference. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMATTR>
    <OMATP>
      <OMS cd="rdf" name="literal_type"/>
      <OMA>
        <OMS cd="rdf" name="resource"/>
        <OMSTR>&lt;http://www.w3.org/2001/XMLSchema#string&gt;</OMSTR>
      </OMA>
    </OMATP>
    <OMSTR>This is a string.</OMSTR>
  </OMATTR>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics>
  <cs>This is a string.</cs>
  <annotation-xml cd="rdf" encoding="literal_type"><apply><csymbol cd="rdf">resource</csymbol><cs>&lt;http://www.w3.org/2001/XMLSchema#string&gt;</cs></apply></annotation-xml>
 </semantics>
</math>
Attrib([ literal_type resource ( "<http://www.w3.org/2001/XMLSchema#string>" ) ], "This is a string." )
"This is a string."{rdf.literal_type -> rdf.resource("<http://www.w3.org/2001/XMLSchema#string>")}
This is a string.

literal_lang

Role:
semantic-attribution
Description:

A symbol to be used within an OpenMath attribute to specify the language of an RDF literal. The annotation value should be an OpenMath string.

Example:
An example to show the representation of the RDF literal "This is an English text."@en. 
<OMOBJ xmlns="http://www.openmath.org/OpenMath" version="2.0" cdbase="http://www.openmath.org/cd">
  <OMATTR>
    <OMATP>
      <OMS cd="rdf" name="literal_lang"/>
      <OMSTR>en</OMSTR>
    </OMATP>
    <OMSTR>This is an English text.</OMSTR>
  </OMATTR>
</OMOBJ>
<math xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics><cs>This is an English text.</cs><annotation-xml cd="rdf" encoding="literal_lang"><cs>en</cs></annotation-xml></semantics>
</math>
Attrib([ literal_lang "en" ], "This is an English text." )
"This is an English text."{rdf.literal_lang -> "en"}
This is an English text.
